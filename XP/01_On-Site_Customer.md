# XP Practice: On-Site Customer

## Project

IACodeEditors

**Owner (Accountable):** Daniel Nuñez\
**Authors:** Daniel Nuñez, Ian (AI Collaborator)

## Summary

This project adapted the XP On-Site Customer practice to an AI-assisted
architecture workflow.

The human owner retained accountability, priorities and final decisions,
while the AI collaborator remained continuously available to discuss
ideas, review alternatives, propose experiments, synthesize findings and
generate living documentation.

## Roles

### Human Owner

-   Defines goals.
-   Prioritizes work.
-   Makes final decisions.
-   Validates architectural conclusions.

### AI Collaborator

-   Reviews ideas.
-   Challenges assumptions.
-   Designs experiments.
-   Produces documentation.
-   Preserves architectural consistency.

## Observed Workflow

Idea → Discussion → Experiment → Evidence → Architecture → Documentation
→ Next Iteration

## Key Findings

-   Immediate availability significantly reduced feedback latency.
-   Architectural decisions became evidence-driven instead of
    assumption-driven.
-   Documentation evolved continuously with the project.
-   Small experimental iterations produced rapid learning.
-   Human accountability remained explicit throughout the process.
-   GitHub ecosystem characterization demonstrated the effectiveness of
    this workflow across SCM, Platform, Copilot, Enterprise policies,
    Linux and Windows implementations.

## Benefits

-   Continuous architectural peer review.
-   Living documentation.
-   Faster hypothesis validation.
-   Better preservation of design rationale.
-   Reduced context switching.
-   Consistent architectural language.

## Adaptation of XP

Traditional:

Customer ↔ Development Team

Adapted:

Human Owner ↔ AI Collaborator ↔ Evidence ↔ Architecture ↔ Implementation

## Conclusion

The adapted On-Site Customer practice proved highly effective for
architectural discovery and design. The AI enhanced availability and
analysis while the human remained accountable for every architectural
decision, making this practice a foundational element for IACodeEditors.
